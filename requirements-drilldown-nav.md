## 1. 概要

### 1.1 目的

第1次機能追加で「タブ切替式の統合管理View」を導入したが、タブ式は「Project → Ticket → Todo」という**階層構造に沿った日常動線**と合っていない。本機能は管理Viewを**ドリルダウン型ナビゲーションUI**に作り替え、次の動線を実現する:

```
Dashboard ────────────┐
                      ▼
プロジェクト一覧 → プロジェクト詳細 → チケット詳細
(Goalグルーピング)   (チケット一覧      (Todo一覧)
                     +Todo一覧)
```

- 一覧から項目を選ぶと同じView内で詳細ページへ遷移する(画面の行き来はパンくず/戻るで)
- Dashboardは現状のまま維持し、Widget項目クリックから該当詳細ページへ直接ジャンプできるようにする

### 1.2 既存機能との関係(確定事項)

| 対象 | 扱い |
|---|---|
| タブ式統合管理View(pos-manage) | **ドリルダウン型に作り替える**(置き換え)。インライン編集セル・フィルタ・RowMenu等の部品は全面流用 |
| Dashboard | 現状維持+Widget項目から詳細ページへの遷移を追加 |
| Preview(右ペイン詳細編集) | 現状維持(併存。右ペインでの編集ユースは残る) |
| Kanban / Search / Timeline | 現状維持(本書のスコープ外) |

### 1.3 スコープ外(将来拡張)

- Goal詳細ページ(Goalの深掘りはPreview/ノートで対応継続)
- View内履歴の永続化(Obsidian再起動後の画面状態復元)
- 一覧のD&D並び替え

## 2. 画面構成

### 2.1 ナビゲーションモデル

- 管理View(pos-manage)内部を**画面スタック**で管理する: `プロジェクト一覧`(ルート)→ `プロジェクト詳細` → `チケット詳細`
- 画面上部に**パンくずリスト**を常設: `プロジェクト ▸ 住宅購入 ▸ 住宅ローン比較`。各要素クリックでその階層へ戻る
- 「← 戻る」ボタン(パンくず先頭)も併設。モバイルでも操作しやすい位置に
- 遷移してもView(ペイン)は増えない(同一ItemView内での画面切替)

### 2.2 プロジェクト一覧(ルート画面)

- **Goalごとのセクションに分けて表示**(Goal未設定のProjectは末尾「未分類」セクション)
  - Goalセクションヘッダ: Goal名+status+配下Project数。クリックで折りたたみ
- 各Project行: Title / Status / Priority / Progress(バー)/ Due / Blockerバッジ(既存セル部品で**インライン編集可**)
- 行クリック(Title部)で**プロジェクト詳細へ遷移**
- フィルタバー(既存流用): キーワード / status / priority / 期間 / tags / labels / archived表示
- 「+ 新規プロジェクト」ボタン(Goalセクション内のボタンはそのGoalを初期値に)
- 行メニュー(⋮): ノートを開く / Previewに表示 / Archive / 削除

### 2.3 プロジェクト詳細ページ

構成(上から):

1. **ヘッダ**: パンくず+Project名(インライン編集)+主要プロパティ(status/priority/due/progress — 既存セルでインライン編集)+「ノートを開く」ボタン
2. **チケット一覧セクション**: 配下Ticketの一覧(Title/Status/Priority/Progress/Due、インライン編集可)
   - 行クリックで**チケット詳細へ遷移**
   - 「+ 新規チケット」(このProjectを初期値に)
   - 行メニュー: ノートを開く / Previewに表示 / 昇格(→Project)/ Archive / 削除
3. **Todo一覧セクション**: **表示範囲切替トグル付き**
   - デフォルト「直下」: このProjectノート直下のTodoのみ
   - 「すべて」: 直下+配下全Ticketの未完了Todoを集約(各行に所属Ticket名バッジ。クリックでそのチケット詳細へ)
   - 完了トグル / インライン編集(text/due/priority)/ 削除 / 昇格 / 完了済み表示トグル
   - 「+ Todoを追加」(直下Todoとして追加。既存addToSection)

### 2.4 チケット詳細ページ

1. **ヘッダ**: パンくず+Ticket名(インライン編集)+主要プロパティ(status/priority/due/progress)+Blockers表示+「ノートを開く」
2. **Todo一覧セクション**: このTicketのTodo(完了トグル/インライン編集/削除/昇格/完了済み表示トグル/「+ Todoを追加」)

### 2.5 Dashboardからの遷移

- Active Projects Widget の項目クリック → 管理Viewを開き**該当プロジェクト詳細**を表示
- Active Tickets / Overdue / Blocked / Review Needed Widget のTicket項目クリック → **該当チケット詳細**
- Todo項目(Today's Todo / Overdue のTodo)クリック → **所属ノートの詳細ページ**(ticket→チケット詳細、project→プロジェクト詳細、inbox→従来通りノートを開く)
- 従来の「ノートを開く」動作は各Widgetの行メニューまたは修飾キークリック等で残す(方式は設計で確定)

## 3. 機能要件

### 3.1 操作(既存UIファースト機能の維持)

- 第1次機能追加で実現した操作(インライン編集/status変更/Todo完了・追加・削除/昇格/Archive/削除/新規作成)は、ドリルダウン型の各画面でも**同等に使えること**(機能後退させない)
- 書き込み経路は既存のEntityFieldService / EntityService / TodoServiceをそのまま使う(新しい書き込みロジックを追加しない)

### 3.2 SavedView

- プロジェクト一覧画面のフィルタ+ソートをSavedViewとして保存・復元できる(viewMode: "manage" を流用)
- 既存の `tab` フィールド付きSavedView(旧タブ式で保存されたもの)は、読み込んでもエラーにならないこと。旧tab情報の詳細な互換マッピングは設計で確定する
- list / kanban のSavedViewは影響を受けない

### 3.3 状態保持

- View生存中は画面スタック・各画面のフィルタ状態・Todo範囲トグルを保持する(詳細→一覧へ戻ってもフィルタが消えない)
- 別のプロジェクト詳細へ移った場合、Todo範囲トグル等のページ内設定はデフォルトに戻ってよい

### 3.4 機能制限モード

- Tasks/Dataview無効時: Todo一覧セクションを非表示+案内バナー(既存E001準拠)。プロジェクト/チケットの一覧・詳細・編集は動作継続

### 3.5 更新反映

- 外部編集・他画面での操作による変更は、`index-updated` 購読(既存100msデバウンス)で表示中の画面に自動反映される
- 表示中のEntityが削除/Archiveされた場合: 詳細ページは一つ上の階層へ自動で戻る+Notice

## 4. 非機能要件

- 画面遷移は再マウントなしのSvelte内部切替とし、体感遅延なし(データはIndexStore読み出しのみ)
- 既存のパフォーマンス機構(SelfWriteGuard/デバウンス/楽観的更新)を維持
- モバイル: タップのみで全動線が成立(パンくず/戻る含む)。行のタップ領域は十分に確保
- i18n: 全文言 `i18n/ja.ts` 集約
- 既存テスト(168件)を壊さない

## 5. 受け入れ基準

1. 管理Viewを開くとプロジェクト一覧(Goalグルーピング)が表示される
2. Project行クリックでプロジェクト詳細(チケット一覧+Todo一覧)へ遷移できる
3. Ticket行クリックでチケット詳細(Todo一覧)へ遷移できる
4. パンくず/戻るで任意の上位階層へ戻れ、フィルタ状態が保持されている
5. プロジェクト詳細のTodo一覧が「直下/すべて」で切り替えられ、「すべて」では所属Ticket名が表示される
6. DashboardのWidget項目クリックで該当詳細ページへ直接遷移できる
7. 各画面でインライン編集・status変更・Todo操作・新規作成・昇格・Archive・削除が第1次機能追加と同等に動作する
8. 表示中Entityの削除/Archive時に上位階層へ自動で戻る
9. SavedViewの保存・復元がプロジェクト一覧で機能し、旧形式SavedViewでもエラーにならない
10. Tasks/Dataview無効時、Todoセクションのみ無効化され、他は動作する
11. 外部編集が表示中画面へ自動反映される
12. モバイルでタップのみで全動線が操作できる
