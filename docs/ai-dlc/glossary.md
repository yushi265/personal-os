# AI-DLC 用語集（SSoT）

> AI-DLC フローで使う中核用語の**唯一の正本（SSoT）**。各ルール（`.claude/rules/`）・スキル（`.claude/skills/`）・
> テンプレ（`docs/spec/_TEMPLATE/`）・ガイドはここを参照し、**定義を二重に書かない**。語義がずれたら本書を先に直す。
> 関連: [ai-dlc-flow](../../.claude/skills/ai-dlc-flow/SKILL.md)（フロー手順）・[ai-dlc-flow-guide.md](../ai-dlc-flow-guide.md)（読み手向けガイド）・[risk-tiers.md](../../.claude/rules/risk-tiers.md)（Tier）。

## 用語定義

| 用語 | 定義 | 例（プロジェクト依存） |
|---|---|---|
| **ビジネスインテント**（Business Intent） | 作りたいことの源（生の要望）。AI が要件へ変換する起点 | 機能のかたまり。おおむね **1 BC** に対応 |
| **BC**（Bounded Context・境界づけられたコンテキスト） | 1 つのモデル・意味が一貫して通用するドメインの境界。**ユニットを切り出す母体** | コード上のモジュール境界（例: サービス層のドメイン別モジュール）と対応。BC の一覧・粒度は要件側で定義 |
| **ユニット**（Unit of Work） | 実装の単位。従来の **Epic の代替**。**設計＋コードを end-to-end で完結**させる。**1 BC から複数生まれる** | **データ層から表示層まで縦に貫く 1 機能**（例: データ → サービス → 表示）＝ **1 チケット**。単一レイヤーで完結する場合もある（CLI・ライブラリ等） |
| **ボルト**（Bolt） | 1 ユニットを作る**短サイクル**（数時間〜数日）。従来の **Sprint の代替**。`ai-dlc-flow` を 1 周 | **1 ユニット = 1〜数ボルト**（同ブランチで DoD まで重ねる）。<br>※ **PR の単位はハーネスで固定しない**（[後述](#pr-の単位について)） |
| **ステージ**（Stage） | ボルト内の工程（7 段） | 要件 → spec → TDD → 静的解析 → セルフレビュー → コミット |
| **ゲート**（Gate） | ボルト内で**人間が承認する点**（最大 3 つ） | Gate 1（要件＋Stage 宣言）/ Gate 2（spec＝契約・委任可）/ Gate 3（コミット） |
| **Gate 2 委任**（Gate 2 Delegation） | **Gate 1 で人間が明示的にオプトインした場合に限り**、Gate 2 を承認待ちなしへ降格する運用。spec は**必ず作成**し要点サマリも**必ず提示**する（変わるのは**承認待ちの有無だけ**）。提示後は承認を待たず Stage 3+4 へ進み、**Gate 3 で実装とまとめて事後確認**を受ける。デフォルトは非委任・**AI から勧めない**（中立提示のみ） | 対象は **Tier 1/2 の spec 実行ボルト**（Tier 3 は spec を作らないため対象外）。**Gate 3 は委任不可**。ティア再宣言（昇格）時は委任もリセット。運用詳細は [risk-tiers.md](../../.claude/rules/risk-tiers.md) |
| **リスクティア**（Tier） | 変更リスクの階級。**ゲートの深さ**を決める | Tier 1（変更コストが高い領域: スキーマ・認証認可・レイヤー間 IF・ロール体系・公開 API 等）〜 Tier 3（typo・doc 等） |
| **SSoT**（Single Source of Truth・信頼できる唯一の正本） | ある情報の**正本が 1 箇所だけ**に存在する状態。ずれたら正本（docs 側）を先に直す | 要件／設計／実装仕様の 3 層 |

## 階層関係

```
ビジネスインテント（生の要望）
  └─ BC（ドメイン境界。ユニットを切り出す母体）
       └─ ユニット（Unit of Work = 1 チケット = データ層→表示層の 1 機能）
            └─ ボルト（ai-dlc-flow を 1 周。1 ユニット = 1〜数ボルト）
                 └─ ステージ（7 工程） … 要所に ゲート（人間承認・深さは Tier が決める）
```

## PR の単位について

- **ボルト・ユニットは「作業分解と工程」の単位であり、PR 分割の規約ではない。**
- **PR の粒度はハーネスで固定しない。** コミット／PR の対象は **Gate 3（コミット承認）で人間がその都度決める**。
- PR を作る際の目安（行数上限ではなく定性基準）は [task-and-pr.md](../../.claude/rules/task-and-pr.md) を参照。
