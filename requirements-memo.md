## 1. 概要

### 1.1 目的

Ticket / Project の作業中に「いつ・何があったか」を残す**タイムスタンプ付きメモ**(Jiraのコメント風)を、詳細画面からワンアクションで追加できるようにする。

- 銀行から回答きた、方針変えた、待ち理由が変わった — such 経緯を日時付きで積んでいける
- AI Export時にも「経緯」として文脈を提供できる(Markdownに残るため)

### 1.2 基本原則(確定事項)

| 項目 | 決定 |
|---|---|
| 対象 | **Ticket + Project**(両詳細画面にメモセクション) |
| 編集・削除 | **両方可**(編集時は元の日時を維持、内容のみ更新) |
| 表示順 | **新しい順**(UI表示のみ逆順。ノート内は時系列で末尾追記) |
| 入力の手軽さ | **ワンアクション追加**: 入力欄は常設、書いてEnter(改行はShift+Enter)で即追加。日時は自動付与。ダイアログ・確認なし |

- Markdown First: メモはノート本文の専用セクションに保存(独自DBなし)。手書きでも同じ形式で書ける
- 長文の自由記述は従来通りノート本文(メモは短い経緯ログ、本文は思考の作業場、という役割分担)

## 2. データ形式

### 2.1 保存形式

ノート本文の `## Memo` セクション配下に、日時見出し+本文のブロックで保存する:

```markdown
## Memo

### 2026-07-04 16:30
銀行から回答きた。金利は1.2%で確定

### 2026-07-04 18:02
比較表に反映済み。
次は引っ越し業者の選定に移る
```

- 見出しは `### YYYY-MM-DD HH:mm` 固定形式(この形式に一致する見出しのみメモとして解釈)
- メモ本文は複数行可(次の見出しまたはセクション終端まで)
- `## Memo` セクションがなければ初回追加時に自動作成(既存 `## Todo` 自動作成と同じ流儀)
- 同一分内の複数メモも許容(見出し重複可。編集・削除は見出し+本文の照合で特定)

### 2.2 解釈の互換性

- ユーザーが手書きで同形式のブロックを追加した場合も、UIのメモ一覧に表示される
- 形式に一致しない記述(自由な文章など)が `## Memo` セクション内にあっても壊さない(UIには出さず、書き込み時も保全)

## 3. UI要件

### 3.1 メモセクション(Ticket詳細 / Project詳細)

- 配置: Todo一覧セクションの下に「メモ」セクション
- **入力欄を常設**(セクション先頭): placeholder「メモを追加…」、Enterで追加(Shift+Enterで改行)。追加後は入力欄クリア+一覧に即反映
- 一覧: 新しい順。各メモは「日時(muted・小さめ)+本文」のカード風表示
- 各メモの操作(ホバー/モバイルは常設ボタン): 編集(その場でtextarea化→Enter確定/Escキャンセル)・削除(確認あり)
- 0件時: 軽い空メッセージ(入力欄は表示したまま)
- 件数が多い場合: 初期表示は直近N件(例: 10件)+「もっと見る」

### 3.2 Preview(右ペイン)

- Previewにも同じメモセクションを表示する(共通部品化)。編集・追加・削除も同等に可能

### 3.3 その他

- ActivityLogへの記録は**しない**(メモ自体が経緯ログのため二重記録を避ける)
- メモ追加/編集/削除はTasks/Dataviewに依存しない(capability無関係に動作)

---

## 4. 非機能要件

- メモの読み出しはIndexStoreに保持せず、詳細画面表示時に本文をオンデマンド読込(cachedRead)する(既存「本文はインデックスに持たない」原則の維持)
- 書き込みは `vault.process`(アトミック)経由。編集・削除は既存editLine同様「期待内容との照合→不一致ならNotice+再読込」方式
- 書き込み後の画面反映は既存 `index-updated` フロー+ローカル再読込で遅延なく
- UIで書いたメモは手書きと同一フォーマット(Git差分の一貫性)
- i18n集約 / TypeScript strict / 既存テスト(199件)を壊さない

---

## 5. 受け入れ基準

1. Ticket詳細・Project詳細・Previewにメモセクションが表示される
2. 入力欄に書いてEnterするだけでメモが追加され、`## Memo` セクションに `### YYYY-MM-DD HH:mm` +本文で書き込まれる
3. `## Memo` セクションがないノートでも初回追加で自動作成される
4. メモ一覧が新しい順で表示され、日時が読める
5. 既存メモの編集(日時維持・内容更新)と削除(確認あり)ができる
6. 複数行メモが正しく保存・表示される(Shift+Enterで改行入力)
7. 手書きで追加した同形式のメモもUIに表示される
8. 形式外の記述が `## Memo` 内にあっても壊れず保全される
9. 外部編集後、表示中のメモ一覧が自動で最新化される
10. Tasks/Dataview無効時でもメモ機能は動作する
