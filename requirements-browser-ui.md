## 1. 概要

### 1.1 目的

Personal OSの管理UI(一覧・詳細・Todo・メモ)を**ブラウザから操作できる**ようにする。

- Obsidianのペインより広い画面・自由なウィンドウ配置で操作したい
- Obsidianのテーマ制約から離れ、**モダンなUI(shadcn系デザインシステム)**で操作性・見た目を作り込みたい

### 1.2 アーキテクチャ方針(確定)

```
ブラウザ(SPA)⇔ HTTP + WebSocket(localhost)⇔ プラグイン内蔵サーバー ⇔ 既存Services / IndexStore
```

- **単一書き込み経路の維持**: Vaultへの読み書きはすべて稼働中のObsidian(既存Services)経由。ブラウザ側はAPIクライアントに徹し、ファイルを直接触らない(データ整合性・SelfWriteGuard等の既存機構をそのまま活かす)
- **Obsidian内UIは現状維持**: Svelte+ネイティブ質感のUIはそのまま。ブラウザUIは並行して存在する第2のフロントエンド
- Obsidian起動中のみ利用可能(サーバーはプラグインのライフサイクルに従う)

### 1.3 スコープ外(将来拡張)

- モバイルObsidianでのサーバー起動(Node API不可のため。プラグイン本体は従来通り動作)
- LAN/リモートアクセス(初版はlocalhostのみ。将来やる場合はTLS・認証強化とセットで別途要件化)
- ブラウザからのノート本文編集(本文はObsidianで書く。従来の役割分担を維持)
- Kanban / Timeline / Advanced Search / AI Export のブラウザ版(初版は管理系に集中)
- オフライン動作・PWA化

## 2. サーバー要件(プラグイン側)

| 項目 | 要件 |
|---|---|
| 有効化 | **設定でopt-in(デフォルトOFF)**。ONの間だけサーバー起動。デスクトップ限定(モバイルでは設定項目自体を非表示) |
| バインド | `127.0.0.1` のみ(外部からアクセス不可) |
| ポート | 設定可能(デフォルト例: 27124近辺の空きポート)。使用中なら自動で次を試行+設定画面に実ポート表示 |
| 認証 | 初回有効化時に生成するトークン(data.json保存)。全APIリクエストで検証。設定画面に「ブラウザで開く」ボタン(トークン付きURLを開く)+トークン再生成ボタン |
| API | REST(一覧・詳細・作成・更新・削除・昇格・Archive・メモCRUD・Todo操作)+ WebSocket(`index-updated`等の変更プッシュ) |
| 静的配信 | ブラウザUIのビルド成果物をプラグインが同梱・配信(外部CDN不使用 — ローカル完結) |
| 停止 | 設定OFF・プラグインunload・Obsidian終了で即停止 |
| ログ | 起動/停止/認証失敗をconsoleへ。Noticeで起動時にURL通知(設定でOFF可) |

- コミュニティプラグイン公開時の審査を意識し、「ネットワーク機能はopt-in・localhost限定・目的明記」をREADME/設定画面に明示する

## 3. ブラウザUI要件

### 3.1 技術スタック

- **shadcn系デザインシステムを採用**(React + shadcn/ui か、Svelte + shadcn-svelte かは設計工程で選定。判断材料: 既存domain型の再利用性・バンドルサイズ・保守の手間)
- Tailwind CSS / ダークモード対応(OS設定追従+手動切替)
- ビルドはVite。リポジトリ内 `webapp/`(仮)として独立ワークスペース化し、プラグイン本体のビルドと分離

### 3.2 画面構成(Obsidian内ドリルダウンUIと同等の情報構造)

| 画面 | 内容 |
|---|---|
| ホーム(Dashboard相当) | Today's Todo / Overdue / Review Needed / Blocked / Active Projects のサマリカード。項目クリックで該当詳細へ |
| プロジェクト一覧 | Goalグルーピング+折りたたみ、フィルタ(コンパクトなツールバー — 現行の「フィルタ▾+バッジ」方式を踏襲)、ソート |
| プロジェクト詳細 | ヘッダ(プロパティ編集)+チケット一覧+Todo一覧(直下/すべて切替)+メモ |
| チケット詳細 | ヘッダ+Blockers+Todo一覧+メモ |
| クイック追加 | グローバルなQuick Add(Todo追加。保存先選択・📅・priority) |

- ナビゲーション: パンくず+ブラウザの戻る/進むボタンが機能する(URLルーティング: `/projects/:path` 等)
- **ノートを開く**: 各所に「Obsidianで開く」ボタン(`obsidian://open` URLスキーム)

### 3.3 操作(Obsidian内UIと同等 — 機能後退させない)

- インライン編集(title/status/priority/due/親/tags/labels/blockers)
- Todo: 完了トグル・追加・編集・削除・昇格
- メモ: 追加(Enter送信・IME対応)・編集・削除
- Entity: 新規作成・昇格・Archive・削除(確認ダイアログ)
- 楽観的更新+失敗時ロールバック+トースト通知(shadcnのtoast)

### 3.4 リアルタイム同期

- WebSocketで変更をプッシュ受信 → 表示中データを自動更新(Obsidian側での編集・手書き編集が即ブラウザに反映)
- 切断時: 画面上部に「接続が切れました(Obsidianが起動しているか確認してください)」バナー+自動再接続
- ブラウザ側操作の反映は「API応答で即時+プッシュで収束」(既存の楽観的更新パターンと同じ思想)

## 4. 非機能要件

- 応答性: 一覧・詳細表示はIndexStore由来のAPIで体感遅延なし(500件規模)
- セキュリティ: localhost限定+トークン必須。トークンなしアクセスは401。CORSは同一オリジンのみ
- 既存機能への影響ゼロ: サーバーOFF時のプラグイン挙動は完全に従来通り。既存テスト(226件)を壊さない
- ブラウザ対応: 最新のChrome/Safari/Firefox/Edge。スマホブラウザは初版ではベストエフォート(レスポンシブは考慮するが最適化は将来)
- i18n: ブラウザUIも日本語文言を一元管理(既存ja.tsの共有 or webapp側リソース — 設計で確定)

## 5. 受け入れ基準

1. 設定でサーバーをONにすると、表示されたURLをブラウザで開いてUIが使える(OFFなら一切起動しない)
2. トークンなしのアクセスは拒否される。localhost以外からは到達できない
3. プロジェクト一覧(Goalグルーピング)→プロジェクト詳細→チケット詳細のドリルダウンと、ブラウザの戻る/進むが機能する
4. Obsidian内UIでできる管理操作(§3.3)がブラウザからも同等にできる
5. ブラウザでの変更がObsidian側(ファイル・UI)に即反映され、逆方向も自動反映される
6. 「Obsidianで開く」でObsidianの該当ノートが開く
7. shadcn系のUI(ダークモード対応)で表示され、Obsidianのテーマ設定と独立して動作する
8. Obsidian終了・プラグイン無効化でブラウザ側に切断バナーが出て、再起動後に自動復帰する
9. サーバーOFF状態で既存の全機能・全テストが従来通り動く
10. モバイルObsidianでプラグインが従来通り動作する(サーバー機能は非表示)

- **工数**: これまでの機能追加1本分以上(サーバー基盤+APIレイヤ+新規フロントエンド一式)。Phase分割前提
