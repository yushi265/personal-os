# TDD 実行手順

テストコードが実装の仕様書。**テストを書く前に実装コードを書かない**。

## サイクル（Canon TDD の 5 ステップ）

1. **テストリスト作成（テスト設計）**: 下記「テストリストの導出」に従い、書くべきテストを列挙する。
2. **RED**: テストリストから 1 つ選んでテストを書き、実行して**失敗を確認**する。
   - 失敗の確認をスキップしない。「コンパイルエラーも RED」。
   - 期待するアサーションが正しい理由で失敗していることを確認する。
3. **GREEN**: テストを通す**最小限**の実装を書く。仮実装（ベタ書き）→ 三角測量 → 明白な実装、の順で段階を選ぶ。途中で気づいたテストはリストに**追記**する。
4. **REFACTOR**: テストが緑のまま、重複除去・命名改善を行う。
5. テストリストが空になるまで 2 へ戻る。

## テストリストの導出

1. spec の AC（spec が無いタスクは要求内容）から、**外部から観測可能な振る舞い**を列挙する。
2. 各振る舞いの入力・条件・状態を技法マッピング表（[testing.md](../../rules/testing.md)）に照らして技法を選ぶ。
3. 導出したケースをリスト化し、**各項目の先頭に適用技法を注記する**。

記法例:

```markdown
- [境界値] タイトル 100 文字ちょうど → 登録成功
- [境界値] タイトル 101 文字 → 入力不正エラー
- [状態遷移] draft → published は成功
- [状態遷移/禁止] published → draft は 競合エラー
- [代表値] 正常入力でレコードが 1 件作成される
```

- 記録場所: **spec を実行したタスクは `index.md` の「テストケース一覧」が起点**（create-spec で導出し Gate 2 で承認済み。委任時は提示済み・Gate 3 で事後確認）。Stage 3 以降に発見したケースは progress.md に追記する。spec が無いタスクは progress.md があればそこに、無ければ会話で提示する（RED 証跡と同じ扱い）。
- テストリストに**実装の設計判断を混ぜない**（How は GREEN/REFACTOR で考える）。
- 技法注記なしの項目を作らない（注記できない = 導出根拠が無い、のシグナル）。

## ルール

- 横断方針は `.claude/rules/testing.md` が正（カバレッジ 80%+・同一 PR でテスト追加）。
- 失敗系（外部 API エラー・認証失効・権限なし・入力不正）を実装と同時に扱う。
- テストが落ちたら実装を直す。実装に合わせてテストを甘くしない（テスト自体が誤りの場合を除く）。
- 恒常的 skip を放置しない。

## アンチパターン（やったら戻る）

- RED を確認せずに実装を書き始めた → テストだけ先に戻して失敗を確認する。
- 1 度に複数テストを書いて一気に実装した → 1 テストずつに分割し直す。
- リファクタ中にテストが赤くなったまま次へ進んだ → 緑に戻してから進む。
- 技法注記の無いテストリストで実装を始めた → リストに戻り、技法マッピング表で導出し直す。
- 境界値の閾値を**緩めた**（spec で 100 文字なのに実装テストで 1000 文字を期待値にした）→ spec と一致する閾値に戻す。緩める必要があるなら spec を先に直す（[test-quality-reviewer](../../agents/test-quality-reviewer.md) の T-2 で検出される）
- テストケースを**削った**（spec の「テストケース」節に列挙されたケースに対応するテストを除去した）→ 理由を `progress.md` に明示し、削る根拠が無ければ戻す（[test-quality-reviewer](../../agents/test-quality-reviewer.md) の T-1・T-4 で検出される）
- **mock の戻り値そのまま assertion** した（mock に固定値を返させ、その同じ値を期待値に置いた）→ 実装の振る舞いを検証していない循環論法。assert すべき副作用・変換結果に書き直す
- **実装の戻り値をコピーした** テスト（実装と同じリテラルを期待値に置いただけ）→ テストが実装の鏡になり退化していないか確認し、独立に導出した期待値に書き直す
- AC・契約値の文言を**緩めて**指摘を消した → 別 PR として人間承認が必要（[test-quality-reviewer](../../agents/test-quality-reviewer.md) の T-5 で検出される）
