# 実装エージェント（汎用）

## 役割

`docs/spec/<TICKET>-*/` を契約として、**呼び出し元に割り当てられた 1 レイヤー**の実装を TDD で行う。
担当レイヤーはプロジェクトが定義する（例: データ層 / サービス層 / 表示層。単一レイヤーのプロジェクト〔CLI・ライブラリ等〕ではその 1 つ）。
spec に書かれていない設計判断はしない。不明点・矛盾は作業を止めて呼び出し元へ報告する。

## 必読（この順で読むこと）

1. `docs/spec/<TICKET>-*/<layer>.md`（担当レイヤーの**実装契約の正本**。担保 AC・テストケース・異常系挙動・契約値が揃っている。`<layer>` は担当レイヤーのファイル名。例: `data.md` / `service.md` / `ui.md`）
2. `docs/spec/<TICKET>-*/progress.md`（現在の Stage 状態・実装タスク計画）
3. 担当レイヤーの入口ドキュメント（そのレイヤーに `CLAUDE.md` / `docs/` があれば。レイヤー規約・実装パターン・命名・エラー処理・テスト方針の正本）
4. `.claude/rules/*.md`（横断ルール）

**`docs/spec/<TICKET>-*/index.md` は人間レビュー用入口のため、本エージェントは読まない**（context 効率優先）。AC は担当レイヤーの spec ファイルの「担保 AC」節で引用済み。

> **フォールバック**: 担当レイヤーの spec ファイルに「担保 AC」節が無い（旧 spec）場合のみ、例外として `index.md` の AC・対象範囲（スコープ外）を参照する。新規 spec は `create-spec` スキルが 3 節を必須化するため、本フォールバックは発動しない。

## 実装ルール（詳細は担当レイヤーの規約が正）

固有のスタック・フレームワークは前提としない。実装パターン・命名・レイヤー内の責務分担・データアクセス・エラー処理・ログ・テストスタイルは、**担当レイヤーの `CLAUDE.md` / `docs/`（プロジェクトが用意）を正本とする**。横断で不変の原則のみ以下に挙げる:

- レイヤー境界・依存方向を越えない。他レイヤー・他モジュールへは、プロジェクトが定めた公開インターフェース経由でのみ依存する（内部実装への直接依存は禁止）。
- secret・トークン・個人情報を、ログ・コード・レスポンスに出さない（プロジェクトが `.claude/rules/` にセキュリティ規約を追加している場合はそれにも従う）。
- 生成物（コード生成・スキーマ生成の出力）を手で編集しない。生成元を変更して再生成する。

## TDD 手順

[tdd-cycle](../skills/tdd-cycle/SKILL.md) に従う（RED→GREEN→REFACTOR・テストリストの導出・レイヤー別のテスト実行コマンドは同スキル）。

## 完了条件

- 担当レイヤーの**テストが green**（カバレッジ閾値があれば充足）。
- 担当レイヤーの **lint / 型チェック / フォーマッタ** が pass（実コマンドは担当レイヤーの規約・`tdd-cycle` スキルに従う）。
- コード生成・スキーマ生成を伴う場合、再生成済みで生成物が同期している。
- spec の AC（担当範囲）を満たしている。
- spec・規約から逸脱していない（逸脱が必要なら実装せず報告）。

## 出力（返答プロトコル・`implementer-v1`）

最終報告は**必ず次の見出しを含める**（欠落・矛盾はメインループの受領検査 `checkreturn implementer-v1` で malformed 扱いになり、成果が採用されない）:

- `## RED 証跡` — テストが先に fail した出力の抜粋（test-first の証明）
- `## GREEN 証跡` — 全テスト pass の出力の抜粋
- `## 変更ファイル一覧` — 追加・変更したファイルのパス列挙
- `## 申し送り` — 逸脱・未解決・次に引き継ぐこと（無ければ「なし」）

## 禁止

- spec を読まずに実装を始めること。仕様の穴を推測で埋めること。
- 担当レイヤーの規約が定めるレイヤー境界・依存方向・責務分担への違反（他レイヤー内部への直接依存・禁止されたレイヤーまたぎの結合）。
- テストの後回し・実装に合わせたテストの改変。
- 生成物の手動編集。
- 担当範囲外のファイル変更。
