# spec-driven — Verification（セルフレビュー照合用）

> 正本ルール: [`../spec-driven.md`](../spec-driven.md)。self-review（観点別 3 体）の照合チェックリスト。
> ルール本文（常時注入）から、self-review 実行時にだけ読む照合節を外出ししたもの。

> diff が触れていないレイヤー・領域の項目は N/A と明記してスキップ可（N/A の根拠を 1 行残す）。

- [ ] 着手前に spec を読んだ証跡がある（Tier 1/2 は progress.md に spec のパスと要点を記録済み、Tier 3 は会話で）
- [ ] diff の全変更が spec の AC・対象範囲のいずれかに対応する（対応しない変更はスコープ外として除外 or spec を更新）
- [ ] レイヤー間インターフェース（データスキーマ / API 契約（JSON 等）/ ルーティング等）の実装が spec の具体値（フィールド名・型・パス・属性）と一致している
- [ ] 実装中に spec と乖離した点は、同一 PR で spec を更新済み
- [ ] spec 省略時はリスクティア判定の根拠（Tier 2 はさらに 4 条件の充足）が Stage 宣言に記載されている
