# testing — Verification（セルフレビュー照合用）

> 正本ルール: [`../testing.md`](../testing.md)。self-review（観点別 3 体）の照合チェックリスト。
> ルール本文（常時注入）から、self-review 実行時にだけ読む照合節を外出ししたもの。

> diff が触れていないレイヤー・領域の項目は N/A と明記してスキップ可（N/A の根拠を 1 行残す）。

- [ ] 実装と同一 PR にテストがある
- [ ] RED を先に確認した証跡がある（progress.md がある場合はそこに、無ければ会話で）
- [ ] spec のテストマトリクス（AC × 種別 × レイヤー）の全行について、対応するテストのファイルパスとテスト名を列挙できる（列挙できない行は未達として Must。spec がある場合）
- [ ] 正常系フロー・異常系・境界値がレイヤー内結合テストにある（E2E ステージは廃止・正常系縦フローもレイヤー内結合で担保）
- [ ] テストリストの全項目に適用技法の注記（`[境界値]` 等）がある（tdd-cycle 実行タスクの場合）
- [ ] 値域・長さ・フォーマット制約のある入力に、境界とその前後のケースがある
- [ ] 複数条件の判定ロジックに、デシジョンテーブル由来の全ルール分のケースがある
- [ ] 状態・ライフサイクルを持つ対象に、有効遷移全件 + 禁止遷移代表ケースがある
- [ ] テストを削減・緩和した場合、その理由が `progress.md` または PR 説明に明示されている（[test-quality-reviewer](../../agents/test-quality-reviewer.md) の T-4 検出）
- [ ] 指摘解消のために AC・契約値・境界値を緩める変更を含まない（含む場合は別 PR として人間承認が必要。[test-quality-reviewer](../../agents/test-quality-reviewer.md) の T-5 検出）
- [ ] テストアンチパターン（`skip` / `.only` / コメントアウト / `expect(true).toBe(true)` / mock 戻り値そのまま assertion）を含まない
