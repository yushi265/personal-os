# Lessons

## 2026-07-04: 主要動線をクリック余白に頼らせない(ドリルダウンナビ N1)

- **何が起きたか**: 一覧→詳細の遷移を「`<tr>` クリック(各`<td>`はstopPropagation)」で設計・実装したが、行はセルで埋まっているため実質クリック可能領域がなく、ユーザーが遷移方法を発見できなかった。タイトルクリックはリネーム編集に割り当てられており、期待(名前クリック=開く)と真逆だった
- **ルール**:
  1. 主要動線(一覧→詳細)は**視覚的にリンクと分かる要素**(タイトルのリンクスタイル+ホバー下線+cursor:pointer)に割り当てる。行余白クリックは補助でしかない
  2. 「クリックで編集」と「クリックで遷移」が同じ要素で競合する場合、**頻度の高い操作(遷移)を素のクリック**に、低頻度操作(リネーム)は明示的なアフォーダンス(ホバーの鉛筆アイコン/メニュー)に逃がす
  3. 設計レビュー時に「初見のユーザーがこの画面で最初にやりたい操作を、どこをクリックすれば達成できるか」を必ず問う(discoverabilityチェック)
  4. UIを新設するPhaseの完了条件に「操作アフォーダンスの明示(ホバー状態・カーソル・アイコン)」を含める。機能が動くだけではUI完成ではない
